What?

Primary

  • Generate 3D conformational ensembles on large, small-compound libraries (tedious, quadratic time)

Secondary

  • Compare & contrast CPU-based methods with newer GPU-based methods of conformation
  • Hybrid method: ligand-based, with attention to Gaussian shape similarity
  • Middle-of-the-road performance between ligand & structural methods

Tertiary

  • Test different library generation parameters, e.g. color (chemical alignment), energy minimization

Why?

  • Exploit GPU parallelism -- standard claim of 100x speedup
  • Use publicly available data from PubChem -- huge stores of data
  • PubChem: quantity over quality
  • Practical applications of methodology include drug design & virtual screening

Complexity of Tasks

  • Not much domain knowledge
  • Reading, understanding papers requires relearning a lot of chemistry knowledge
  • Using a lot of high-level frameworks, isolated from difficult GPU programming, e.g. CUDA/OpenCL
  • Abstraction requires benchmarking
  • Handling large data sets (processing in parallel easy)

Products Delivered

  • Applications for HPC resources, focus research proposals
  • Applications and academic licenses for OpenEye ROCS, ChemAxon
  • Reports, suggestions based on papers read
  • Gathered, sorted data sets & subsets

Computing and Other Knowledge Gained

  • The tremendous paperwork behind research
  • Reinforce basic knowledge of chemistry concepts, conformers
  • Basics of chemical informatics, chemical libraries, and how to handle data
  • Advantages of GPU-based computing techniques
  • Similarity-based NxN clustering (intuitive)
  • Algorithmic solutions to physical world problems -- alternatives to HTS

Tools and Techniques Used and Learned

  • PubChem with various chemical interchange formats and metadata
  • OpeneEye (Fast)ROCS, OMEGA
  • ChemAxon Fragmenter & JKlustor
  • Python, OpenBabel
  • State of Open Source tooling is lackluster
  • Shared computing resources
